ของที่ยังไม่ได้แก้ (ต้องอัปเดตทีหลัง)
==========================================

1. time_logs ของหน้า /student ยังเขียนตรงผ่าน anon key (Supabase client-side)
   - เช็คอิน / เช็คเอาท์ / ลงเวลาย้อนหลัง (self-report) ยังไม่ได้ย้ายไปหลัง
     API ที่มี auth เหมือนฝั่ง dev/manager
   - ผลคือ: ใครก็ตามที่ยิง Supabase REST ตรง ๆ (ไม่ผ่านหน้าเว็บ) จะเช็คอิน/
     เช็คเอาท์แทนนิสิตคนไหนก็ได้โดยไม่ต้องรู้ PIN เลย เพราะ PIN check ทำอยู่
     ที่ client-side JS เท่านั้น ไม่ได้บังคับที่ระดับ API/DB
   - เหตุผลที่ยังไม่แก้: เป็น flow หลักที่นิสิตทุกคนใช้ทุกวัน เสี่ยงพังถ้าแก้
     โดยไม่ทดสอบละเอียด ตกลงกันไว้ว่าจะพักไว้ก่อน

2. ยังไม่มี audit log
   - ไม่มีการบันทึกว่า dev/manager คนไหนแก้/ลบ/อนุมัติอะไรเมื่อไหร่
   - เป็น nice-to-have ไม่ใช่ช่องโหว่ความปลอดภัย

3. RLS (Row Level Security) ยังเป็นระดับ baseline (เปิดใช้ แต่ policy
   อนุญาตทุกอย่างเหมือนเดิม) เพราะหน้า /student ยังต้องใช้ anon key ตรง ๆ
   อยู่ (ข้อ 1) เลยล็อกให้แน่นกว่านี้ไม่ได้จนกว่าจะแก้ข้อ 1 ก่อน

4. ค้างคาจากที่คุยไว้ก่อนหน้า — ไม่เคยยืนยันว่าแก้จริงหรือยัง:
   ตอนเปลี่ยนระบบ token ใหม่ (signed + หมดอายุ 24 ชม.) มีรายงานว่า
   /api/manager/login ขึ้น 401 ตอน login — ยังไม่ได้เช็คสาเหตุต่อ เพราะ
   ผู้ใช้เปลี่ยนไปแจ้งบั๊กอื่นก่อน ถ้ายังเป็นอยู่ต้องตามต่อ

สิ่งที่แก้ไปแล้ว (เผื่ออ้างอิง)
==========================================
ความปลอดภัย:
- ปิดช่องดูรายงาน/PDF นิสิตคนอื่นผ่าน /print และ /print-roster
- ย้าย roster (PII) และ time_logs ของหน้า dev/manager ไปหลัง API ที่มี token
- PIN นิสิตไม่ส่งไปที่ browser แล้ว, ตรวจสอบฝั่ง server, hash ที่เก็บใน DB,
  มี rate-limit/lockout กันเดา PIN (5 ครั้งผิด ล็อก 5 นาที)
- เปลี่ยน token ของ dev/manager เป็นแบบ signed + หมดอายุ 24 ชม. + ผูกกับ
  identity ของแต่ละคน (เดิมเป็นความลับตัวเดียวใช้ร่วมกันหมด ไม่หมดอายุ)
- manager ที่ถูกล็อกแผนกเขียนข้อมูลนิสิตแผนกอื่นผ่าน API ตรง ๆ ไม่ได้แล้ว
- ปิดช่อง /api/announcements ที่เชื่อ header x-mgr-username เฉย ๆ โดยไม่มี
  การยืนยันตัวตนจริง
- self-report (ลงเวลาย้อนหลัง) บังคับต้องมี PIN เสมอ ไม่มีทางบายพาสแม้จะยัง
  ไม่เคยตั้ง PIN เลย (ปิดช่องโหว่จากปุ่ม "แก้ไข" ที่เคยเลี่ยงการเช็คได้)

บั๊ก UI / performance:
- popup ข้อความ error/success เคยถูกซ่อนอยู่หลัง modal (self-report เปิดอยู่
  จะไม่เห็น error เลย) — แก้ให้ลอยอยู่บนสุดเสมอ
- popup จัดกึ่งกลางไม่ถูกเพราะ animation ชนกับ CSS transform ที่ใช้ centering
  — เปลี่ยนวิธี centering ใหม่
- ลงเวลาย้อนหลังซ้อนทับกันได้ (race condition ตอนกดรัว ๆ) — ล็อกไม่ให้กดซ้ำ
  ตั้งแต่ต้นฟังก์ชัน + เปลี่ยนจาก window.alert() เป็น popup ในแอป (มือถือบาง
  รุ่นบล็อก alert() หลัง await เงียบ ๆ)
- ประวัติ/badge สรุปเดือนไม่รีเฟรชอัตโนมัติหลังลงเวลาย้อนหลัง ถ้าไม่ได้เปิด
  แผงประวัติค้างไว้ — เอาเงื่อนไขออก รีเฟรชเสมอ
- ตัวเลื่อนเวลา (wheel picker) กระตุก/กระพริบตอนเลือกเลข ("0 1 0 1") — เจอ
  race condition ระหว่าง animation กับ state sync แก้แล้ว ใช้ร่วมกันทั้ง 3
  หน้า (dev/manager/student) แก้ที่เดียวจบ
- คำว่า "ค้นหา..." ตอนกรอกรหัสนิสิตกระพริบ (animate-pulse ตัดจบก่อนครบรอบ)
  — เปลี่ยนเป็นไอคอนหมุนแทน
- ปุ่มทุกปุ่มทั้งเว็บมี animation เงา/ความสว่างตอน hover ที่ทำให้ browser
  ต้อง repaint ทุกเฟรม รู้สึกหน่วงทั้งเว็บ — ตัดเหลือแค่ transform (เบากว่ามาก)
- roster แถวเกินอันดับ 10 เด้งเข้าจอพร้อมกันหมดตอนโหลด (ไม่มี stagger delay)
  ยิ่งคนเยอะยิ่งหน่วง — ปิด animation ให้แถวที่เกิน 10
- เช็คอิน/ลงเวลาย้อนหลัง หน่วง 1-1.5 วิ เพราะแก้ PIN ให้ปลอดภัยขึ้นทำให้มี
  round-trip เพิ่ม — เอาการเช็ค PIN กับการเช็คอย่างอื่น (เช่นมีลงเวลาค้างไหม)
  มายิงพร้อมกันแทนที่จะรอทีละอย่าง ลดไป 1 round-trip ต่อการกด

หมายเหตุ: dev/manager ทุกคนต้อง login ใหม่หลัง deploy รอบที่เปลี่ยน token
(ของเก่าใช้ไม่ได้แล้วทั้งหมด เป็นผลข้างเคียงที่ตั้งใจ)
